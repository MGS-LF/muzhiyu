## 📜 墓之语 · 完整设定总档（确认版 + 技术选型新增）

---

### 一、游戏基础信息

| 项目 | 确认内容 |
|------|----------|
| **游戏名** | **墓之语**（英文：Tomb Whisper） |
| **副标题** | ——每一句真心话都曾是文明的火种。 |
| **赛道** | AI叙事 + 独立游戏（Metagame） |
| **体验时长** | 约60分钟 |
| **核心机制** | 大地图探索 + UT式字符弹幕战斗 + AI实时诗意叙事 |
| **视觉风格** | 极简线条 + 高反差色彩冲击 + 负空间压迫感 |
| **技术选型** | **纯Web前端（HTML + CSS + JavaScript/TypeScript），不依赖Unity/Godot等任何游戏引擎** |
| **渲染方式** | Canvas 2D原生绘图，全部图形代码绘制，零外部图片素材 |


### 二、技术选型理由

| 维度 | 说明 |
|------|------|
| **黑客松适配** | 无需安装引擎、无需打包、修改即刷新，适合快速迭代演示 |
| **AI接入便利** | 浏览器直接调API（SSE流式输出诗句），无中间层延迟 |
| **纯代码绘制** | 所有视觉为Canvas绘图，完美契合“Vibe Coding”极简美学 |
| **跨平台演示** | 一个HTML文件即可运行，评委用手机/平板/电脑都能打开 |
| **版本控制** | 纯文本文件，Git diff清晰，多人协作无二进制冲突 |
| **部署成本** | 静态托管（Vercel/Netlify/GitHub Pages）免费秒级上线 |


### 三、技术架构预览

| 层级 | 技术方案 |
|------|----------|
| **UI层** | HTML + CSS（极简黑底白字，无框架） |
| **渲染层** | Canvas 2D（所有图形/文字/粒子/弹幕均实时绘制） |
| **逻辑层** | 原生JavaScript/TypeScript（ES Module组织） |
| **状态管理** | 单例Store（词汇库、NPC状态、玩家进度） |
| **AI接入** | Fetch + SSE（流式接收AI生成诗句，逐字打印效果） |
| **音频** | Web Audio API（程序化生成8-bit音效，无外部音频文件） |
| **存档** | localStorage（自动保存进度+词汇收集情况） |


### 四、Canvas绘制内容清单（全部代码生成）

| 元素 | 绘制方式 |
|------|----------|
| 火柴人 | `ctx.arc` + `ctx.lineTo` 绘制线条骨骼 |
| 灵魂方块 | `ctx.fillRect` 填充发光方块 |
| 文字弹幕 | `ctx.fillText` 像素字体渲染 |
| 波纹开场 | `ctx.bezierCurveTo` 动态贝塞尔曲线 |
| 战斗框 | `ctx.strokeRect` + RGB通道分离绘制 |
| 粒子系统 | `ctx.fillRect` 像素点阵 + 透明度动画 |
| NPC线条小人 | `ctx.arc` + `ctx.lineTo` 组合轮廓 |
| 时间之树 | 字符逐字排列 + 旋转动画 |
| 全屏闪 | `ctx.fillRect` 覆盖全屏 + 透明度过渡 |
| 屏幕震动 | Canvas整体偏移变换 |

### 七、性能目标

| 指标 | 目标 |
|------|------|
| 帧率 | 60fps（Canvas 2D纯绘制，无3D开销） |
| 弹幕数量 | 同屏最多200个文字弹幕（轻量fillText） |
| 粒子数量 | 同屏最多500个像素粒子（fillRect小方块） |
| 加载时间 | 0秒（无外部资源，HTML打开即运行） |
| 包体积 | < 1MB（全部代码+数据，无图片/音频文件） |
