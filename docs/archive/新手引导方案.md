# 《墓之语》新手引导方案

> 目标：在 **3D 序幕加载结束、进入 2D 主游戏后**，用「梦境剧情 + 分阶段玩法引导」带玩家完成：理解世界观 → 会移动互动 → 会捡字通关 → 会打两种战斗。  
> 原则：不打断核心节奏；可跳过；可存档；不重复轰炸；与现有 `tutorial` / `flags` / `DIALOGS` / `SCENE_INTROS` 复用。

---

## 十三、实装备注（已落地）

> 教学场为**独立场景** `dream_tutorial`（`src/scenes.js`），不复用 `freeze_center` / `street_01`。  
> 入口：3D 序幕结束后 `startOnboardingAfterIntro()` → `beginDreamOnboarding()`。  
> 流程：梦境对白 → 手机幻象(E) → 捡「言」「语」→ 门禁 → 常规梗鬼战 → 黑客噪声核 → 金色裂隙醒来 → 真实冷冻中心。  
> 醒来时清空教学碎片/击杀记录，避免污染主线。可跳过。二周目默认跳过。  
> 关键文件：`src/systems/onboarding.js`、`src/data/dialogs.js`（`onboarding_*`）、`src/main.js`、`src/game.js`、`src/systems/interact.js`。

## 一、触发时机（你指定的入口）

### 1.1 正确挂载点

| 时机 | 现状 | 引导动作 |
| --- | --- | --- |
| 标题「开始游戏」 | `keheng:startIntro` → `intro_3d.html` | 不动序幕本身 |
| 序幕结束 | `intro:done` / `FROM_INTRO` → `wake_done=true` | **在此之后**启动「梦境收束 + 新手引导」 |
| 非序幕直接进（读档/刷新） | 无 `FROM_INTRO` | **不播梦境长剧情**；仅按缺失 flag 补玩法提示 |

**推荐实现钩子：**

1. `main.js`：`finishIntroOverlay()` 之后，或 `FROM_INTRO` 分支里  
   → `game.startOnboarding({ fromIntro: true })`
2. 普通 `Game` 初始化：`createStartTutorial()` **降级**为 onboarding 的一个步骤，不再一上来整页快捷键墙。

> 你说的「3D 界面加载完后」= **序幕 3D 播完回到主 canvas**，不是地铁维度裂隙 `Level3D`。维度裂隙仍用现有入口对话即可。

### 1.2 与现有苏醒对白的关系

- 现有：无序幕时，移动后会触发 `DIALOGS.wake`。
- 有序幕后：`wake_done` 已置位，**不会再播 wake**。
- 本方案：用 **新对话链 `onboarding_dream`** 替代「二次苏醒解说」，专门讲「手机烂梗 → 失语」的梦境。

## 二、引导总结构（两层）

```
Layer A  剧情引导（一次，可跳过）
   └─ 梦境：时代病 / 烂梗 / 失语

Layer B  玩法引导（分步，可跳过单步，按 flag 进度）
   ├─ B1 移动与互动
   ├─ B2 捡字与通关（碎片 + 门禁概念）
   ├─ B3 常规战斗（诗词/菜单战）
   └─ B4 黑客战斗（仿真弹幕/言锋）
```

两层可重叠：剧情结束后立刻进入 B1；B2–B4 用「情境触发」，不是连续弹 4 个全屏。

## 三、剧情引导（梦境）

### 3.1 叙事目标（30–60 秒可读完）

让玩家在进冷冻仓行动前明白：

1. 这不是普通末日，是 **语言被「跟风内容」掏空**；
2. 人们盯着手机学烂梗，生怕落伍；
3. 话越来越少，最后 **只会复制梗、不会真正说话**；
4. 顾言从「梦」里醒来，目标是在废墟里 **捡回文字、对抗失语**。

### 3.2 建议对白骨架（可写入 `dialogs.js` → `onboarding_dream`）

| 节点 | 说话人 | 内容要点 |
| --- | --- | --- |
| 1 | 系统/旁白 | 你仍在梦里。屏幕光比太阳还亮。 |
| 2 | 路人甲（幻象） | 「今天不学三个新梗，群里就没人理我。」 |
| 3 | 旁白 | 他们以为跟上节奏才是活着。 |
| 4 | 路人乙 | 嘴里只剩半截谐音和口号，说不出自己的名字。 |
| 5 | 顾言 | ……他们不是不会发声，是把「说话」交给了别人的句子。 |
| 6 | 旁白 | 梦碎。冷冻舱的雾还在。你要在醒来的世界里，把字重新捡回来。 |
| 7 | 选项 | `继续（开始引导）` / `跳过引导（仅保留目标提示）` |

**呈现方式：**

- 优先：现有 `startDialog` 全屏对话框（与世界风格一致）。
- 可选增强：对话框期间轻微压暗 + 手机 UI 剪影粒子（`reducedFx` 时关闭）。
- **不要**再叠一层旧的 `createStartTutorial` 快捷键墙；快捷键改成分步教。

### 3.3 跳过与二周目

| 情况 | 行为 |
| --- | --- |
| 新游戏 + 序幕结束 | 播 `onboarding_dream` |
| 玩家选「跳过引导」 | `flags.onboarding_skipped=true`，只设目标文案 |
| 二周目 / 已有通关 meta | 默认跳过剧情，保留极简玩法 tip |
| 读档 | 不重播剧情；按未完成 `onboarding_*` flag 补玩法 |

## 四、玩法引导（分步设计）

### 4.0 状态机建议

新增模块（建议）：`src/systems/onboarding.js`  
挂到 `Game`：`this.onboarding = { step, done:Set }`

**核心 flags（存档）：**

```
onboarding_dream_done
onboarding_move_done
onboarding_pickup_done
onboarding_gate_taught
onboarding_battle_done      // 常规战
onboarding_hack_done        // 黑客战
onboarding_all_done
onboarding_skipped
```

**表现形态（统一三种，避免再发明 UI）：**

1. **短对话**（重要规则，1–3 句）
2. **Toast / showHint**（即时操作提示）
3. **目标条 objective**（始终告诉「下一步做什么」）
4. **可选：右下角小卡片**（仅一步，不挡画面；可复用 tutorial 绘制精简版）

---

### B1 · 移动与互动（冷冻中心）

| 项目 | 内容 |
| --- | --- |
| 触发 | 梦境结束，或 `freeze_center` 且 `!onboarding_move_done` |
| 教什么 | WASD 移动、E 互动、看左上「目标」 |
| 完成条件 | 拾取/互动任意一次（换衣服或开门相关 interact）**或** 走到出口附近 |
| 文案示例 | 「靠近发光物体按 E。先换上衣服，再离开冷冻中心。」 |
| 与现状关系 | 替换/收敛 `createStartTutorial` 全键列表；完整键位仍在设置/关于里 |

**注意：** 现有 objective 已是「换上衣服，离开冷冻中心」——引导只 **解释 HUD**，不另起冲突目标。

---

### B2 · 捡字通关（街道）

| 项目 | 内容 |
| --- | --- |
| 触发 | 首次进入 `street_01` 且 `!onboarding_pickup_done` |
| 教什么 | 地上发光 **汉字碎片**；捡字 = 进度 + 战斗弹药；门禁需要集齐指定字 |
| 完成条件 | `recordChar` 首次成功 **或** 拾取任一 `char_fragment` |
| 强化点 | 首次捡字后 toast：「碎片已收入——左上可看收集进度」 |
| 门禁教学 | 首次撞上 `gate` 且缺字时（`meetsGate` 失败）播 1 句：  
  「此路被污染封锁。集齐需要的字，才能复原诗句通过。」  
  → `onboarding_gate_taught` |
| 通关闭环 | 不强制当场通关；指到「集齐 洲/逑 → 江堤方向」即可 |

**与现状关系：**

- 复用 `SCENE_INTROS.street_01`，改写成更短的 onboarding 文案或由其触发。
- 复用 `DIALOGS` 门禁失败提示，避免两套说法。

---

### B3 · 常规战斗（第一次梗鬼）

| 项目 | 内容 |
| --- | --- |
| 触发 | 现有 `first_geng_intro` 接触点（`!first_geng_intro_done`） |
| 教什么 | 靠近绿影开战；菜单：攻击 / 诗词 / 道具；注意理性（SAN） |
| 完成条件 | 首次 `Battle` 结束（胜/逃/败均可记 `onboarding_battle_done`，胜时额外成功 toast） |
| 文案策略 | **压缩**现有 `DIALOGS.first_geng_intro`，改成「3 句内讲清 + 开战」；详细键位用战斗内 `CONTROL_HINTS.battle*` |
| 战斗内 | phase=`intro` 时底部固定一行：「方向选择 · E 确认 · 注意理性条」 |

**不要**在开战前塞完整快捷键表。

---

### B4 · 黑客 / 言锋战斗（第二种模式）

| 项目 | 内容 |
| --- | --- |
| 触发 | 首次 `useHack === true` 的 `startBattle`（已有 `hack_core_intro`） |
| 教什么 | 这与菜单战不同：实时移动、瞄准、躲避信息弹幕；主题是「在噪音里保住句子」 |
| 完成条件 | 首次 `HackingBattle` 结束 → `onboarding_hack_done` |
| 文案策略 | 扩展/精修 `DIALOGS.hack_core_intro`（3–5 句）；`HackingBattle.phase==='intro'` 显示操作条 |
| 若主线很晚才遇到 hack | 在设置「训练」或体育馆前加 **可选试炼 NPC**（低优先级）；主路径仍以首次真实遭遇为准 |

**两种战斗的对比教学（一句就够）：**

> 「有的敌人用诗句对轰；有的把你拖进手机似的噪声场——两种打法都要会。」

## 五、玩家流程时间线（理想路径）

```
[标题] 开始游戏
   → [3D 序幕] 苏醒/世界观影像（已有）
   → [进入 2D] onboarding_dream（手机烂梗→失语）  ← 本次新增重点
   → B1 冷冻中心：动起来、E、换衣服出门
   → B2 街道：捡字、看目标箭头、理解封锁门
   → B3 第一只梗鬼：常规战斗
   → …主线推进…
   → B4 首次黑客战：第二种战斗
   → onboarding_all_done（之后只保留场景 SCENE_INTROS 轻提示）
```

预估强制阅读时长：

- 梦境：约 40–70 秒（可跳过）
- B1–B2：各 1 次 toast + 目标，不强制停手
- B3–B4：各 1 段短对话 + 战斗内 hint

## 六、UI / 交互规范

| 规则 | 说明 |
| --- | --- |
| 同时只存在 1 个「强引导」 | 对话框 **或** 教程卡片，不叠两层全屏 |
| 可跳过 | 梦境选项跳过；玩法步骤长按 Esc 或设置里「跳过新手引导」 |
| 无障碍 | `highContrast` / `reducedFx` / `colorblind` 下引导控件仍清晰；尊重 reduced motion |
| 文案语气 | 废墟暖金风，短句，少术语；键位用 `CONTROL_LABELS` |
| 不教的内容 | 存档快捷键、调试 F2、无尽模式——放设置/关于 |

## 八、内容草案（可直接给文案）

### 8.1 梦境（摘要版，实装可加长）

1. **旁白：** 你梦见一座不夜的城。人人捧着一块发光的板。  
2. **幻象：** 「不发梗就没有人看见你。」  
3. **旁白：** 他们越学越快，嘴却越来越空。  
4. **顾言：** 当所有人都在说同一个笑话，还有谁在说话？  
5. **旁白：** 笑停了。街上只剩复制、粘贴，和失语。  
6. **系统：** 醒来。废墟还在。去把散落的字捡回来。

### 8.2 玩法一句教

| 步骤 | 一句 |
| --- | --- |
| B1 | 移动靠近光点，按 E。 |
| B2 | 汉字碎片是钥匙，也是弹药。 |
| 门禁 | 缺字过不去；凑齐再回来。 |
| B3 | 绿影开战：选行动，护住理性。 |
| B4 | 噪声场里移动射击，别被烂梗吞掉。 |

## 九、实装里程碑（供你拍板后开工）

### M1 · 剧情挂载（0.5–1 天）
- 新增 `onboarding_dream` 对话与 flag  
- `FROM_INTRO` / `finishIntroOverlay` 后调用

### M2 · B1/B2（1 天）
- 冷冻中心 / 街道情境触发  
- 收敛 `createStartTutorial`  
- 捡字、门禁首次教学  

### M3 · B3/B4（1 天）
- 压缩 `first_geng_intro` / `hack_core_intro`  
- 战斗 intro 条与完成 flag  
- 结束检测  

### M4 · 打磨（0.5 天）
- 二周目/读档/跳过路径  
- 文案过一遍、与 objective 不打架  
- 手动回归：新游戏序幕→出门→捡字→第一战→（能触发则）黑客战
