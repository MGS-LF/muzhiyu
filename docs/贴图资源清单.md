# 《墓之语》贴图与简单动画资源清单

> 用途：本清单汇总项目中所有需要独立贴图 / 材质 / 精灵的 **NPC** 与 **场景元素**，以及与之配套的**简单动画方案**。供美术与程序逐项勾选。
> 状态图例：`[ ]` 未制作　`[x]` 已完成　`[-]` 暂缓/非必需
> 全局美术基调（取自代码 `bgColor` / `atmosphere`）：废墟暗调（暖褐 `#241c14`、近黑 `#1c1a16`），暖色尘埃粒子（橙 `rgba(255,200,130)`、米 `rgba(220,190,130)`），深渊与数据空间用冷蓝。梗鬼系用「绿皮/绿光」表达腐化感。

---

## 零、简单动画实现原则（全局统一）

本清单**不使用逐帧手绘动作 / 繁琐关键帧**。所有动画均为**程序化变换**，由现有 `render`（`W2S` 世界→屏幕投影）在每帧用基础数学实时计算，风格与原 Canvas 渲染完全一致：

- **单张贴图为主**：角色/敌人只做 1 张贴图（或 2 态：如失语者治愈前后），靠变换产生动感。
- **可用变换原语**（组合即可覆盖全部需求）：
  - 浮动/呼吸：`y += sin(t)*2`，`scale = 1 + sin(t)*0.02`
  - 行走：随移动做 `rotate(±3°)` 倾斜，无换帧
  - 攻击/挥击：缩放脉冲 `1.0→1.15→1.0`（~150ms）+ 透明度微闪；武器贴图单独 `rotate(±20°)`
  - 受击：叠加白/红闪（透明度）+ 水平抖动 `±3px`（~200ms）
  - 消失/净化：透明度 `1→0` + 缩放 `1→0.6`，配合既有粒子
  - 治愈：两态贴图透明度交叉淡入 + 上浮 `10px`
  - 发光：`sin(t)` 透明度脉冲叠加发光层
  - 环境：尘埃 `motes` 粒子、光点旋转漂浮、数据河横向滚动——均为参数化循环，非帧动画
- **无骨骼 / 无 IK / 无多帧精灵表**。
