# 新手引导重构方案 · 线性梦境（传说之下式）

> 针对当前反馈：教学太散、剧情生硬、主角缺疑惑、左上角不显示要收集的字。  
> 原则：**一条路、一次只教一件事、对话像角色在想、HUD 永远告诉你下一步与要集的字。**  
> 仍保持：**独立场景 `dream_tutorial`，结束后清空教学进度，再进真实冷冻中心。**

---

## 〇、现状问题（对照）

| 问题 | 现状 | 传说之下式目标 |
| --- | --- | --- |
| 结构 | 小开放场 + 多目标并行，玩家可乱走漏步骤 | **单线走廊**：房间 1→2→3→4→出口，每段锁门，做完才开 |
| 剧情 | 系统旁白直接讲「手机烂梗→失语」 | **顾言先困惑**，通过幻象/对话自己悟到，系统少说教 |
| 操作教学 | toast + 模糊目标文案 | **事件锁死**：不完成当前动作无法前进；对话框内嵌操作提示 |
| 碎片 HUD | 未设置 `objective.progress`，左上只显示弹药 | 捡字阶段强制挂 `progress: { title, chars:[{c,have}] }` |
| 战斗 | 两只怪同屏，顺序可乱 | **先只放常规怪**；打完再刷/开门放黑客终端 |

## 一、整体体验对标（传说之下）

```
开场长对话（可跳过整段教学，但不可跳过单句乱序）
  → 极窄移动区 + 一个发光交互
  → 短对话确认「懂了」
  → 门开，进入下一段
  → ……
  → 两种战斗各一次（必打/可宽恕，但必须经历）
  → 醒来
```

**关键差异：**

1. **线性**：不是「探索教学场」，是「过场走廊」。
2. **门禁是剧情门，不是开放谜题**：缺字时箭头指碎片，不让玩家猜。
3. **一次只出现一个可交互重点**（高亮 + 其它交互暂时禁用或未生成）。
4. **战斗前必有 2–4 句对话**，打完立刻有反应，再给下一步。

## 二、场景结构（重做 `dream_tutorial` 布局）

把当前 1400×900 半开放图改成 **4 个串联房间**（仍一个 scene id，用墙隔断 + 单向门）：

```
[R0 序幕厅] --门A--> [R1 捡字廊] --门B--> [R2 常规战室] --门C--> [R3 言锋室] --裂隙--> 醒来
```

| 房间 | 尺寸感 | 唯一目标 | 解锁条件 |
| --- | --- | --- | --- |
| R0 序幕厅 | 小方厅 | 听完开场 + 靠近手机幻象按 E | 开场对话结束 |
| R1 捡字廊 | 窄长廊 | 拾取「言」「语」两枚 | 两字齐 → 门B 开 |
| R2 常规战室 | 中等 | 遭遇弱梗鬼，完成一次菜单战 | 胜/宽恕/败后仍推进（败则弱化再战一次或直接放行并说明） |
| R3 言锋室 | 中等 | 接触噪声核，完成一次黑客战 | 结束后出现金色裂隙 |
| 出口 | 裂隙 | 醒来 | 两种战斗 flag 都 true |

**生成规则：**

- 玩家始终只能看见/走到**当前房间**（后门可关死，或前进后封回）。
- 箭头 `objective.target` 始终指向**当前房间唯一关键物**。
- 黑客怪 **R2 结束前不存在**（`loadScene` 后动态 `enemies` 过滤，或 `_cond` + spawn）。

## 三、剧情改写原则（去生硬）

### 3.1 角色立场

| 角色 | 作用 |
| --- | --- |
| **顾言** | 主角视角：困惑、否认、试探、逐渐明白 |
| **幻影/路人** | 只说半截梗、不解释世界观 |
| **系统** | 少用；仅在「操作说明」与「梦将醒」时出现，且用括号短句 |
| **禁止** | 开场就用系统句把「手机→失语→梗鬼」讲完 |

### 3.2 叙事节拍（建议对话量）

**开场（R0，约 6–8 句，可选项跳过整段教学）**

1. 顾言：……这是哪儿？冷冻舱呢？我明明……  
2. 顾言：光太亮了。像无数块屏幕叠在一起。  
3. 幻影（不回头）：「今天不发三个梗，群里就当我不存在。」  
4. 顾言：等等——你说的是人话吗？还是……口号？  
5. 幻影：只剩「绝绝子」「YYDS」……说不出自己的名字。  
6. 顾言：……他们不是嗓子坏了。是句子被掏空了。  
7. 系统：（这是梦。向前走，你会需要「字」。）  
8. 选项：`继续梦` / `跳过教学醒来`

**手机幻象 E 后（2–3 句）**

- 顾言：他们把「说话」外包给屏幕了……  
- 系统：（地上有碎掉的字。靠近，按 E 拾取。）

**捡到第一字**

- 顾言：……「言」？它烫手，却像是真的。  
- 左上角进度亮起（见第四节）

**两字齐、门开**

- 顾言：绿雾退了。前面还有东西在动……  
- 系统：（靠近绿影会开战。菜单战：选行动，护住理性。）

**常规战结束**

- 顾言：用完整的句子，它就怕了……？  
- 系统：（还有一种战场：噪声会把你拖进逻辑空间。）

**黑客战结束**

- 顾言：有的敌人用诗句对轰，有的把你拽进手机似的噪声里。  
- 系统：（裂隙开了。醒来后，这些只会变成习惯——字要自己捡。）

**醒来（衔接冷冻中心，3 句）**

- 系统：铁锈与霉。这是现实。  
- 顾言：……梦吗？可指尖还记得「言」「语」。  
- 系统：（先换衣服，再出冷冻中心。目标在左上角。）

### 3.3 文案禁忌

- 不说「这是教学场，你要学会两种战斗」这种 meta 直球（可保留在**跳过选项旁**小字）。  
- 不一次丢超过 1 条操作说明。  
- 操作说明用系统括号句，叙事用顾言/幻影。

## 四、左上角碎片进度（必做）

### 4.1 原因

HUD 只在 `game.objective.progress` 存在时画「碎片 · 标题 + 字格」。  
当前 onboarding 只写了 `objective.text`，**没有 `progress`**，所以左上角看不到要收集的字。

### 4.2 规范

从 **R1 开始**到门B打开前：

```js
this.objective = {
  text: '拾取发光的字：言、语',
  done: false,
  target: { x: 下一碎片x, y: 下一碎片y }, // 未集齐的第一枚
  progress: {
    title: '梦境·句子',
    chars: [
      { c: '言', have: collectedAll.includes('言') },
      { c: '语', have: collectedAll.includes('语') },
    ],
  },
};
```

- 每次 `pickup_char` / `recordChar` 后 **刷新 `progress.chars[].have`**。  
- 两字齐后：`progress` 可保留打勾 1 秒，再切到下一目标（战斗），或清空 progress 只留 text。  
- 正式主线的 `refreshObjective` 不要覆盖梦境 objective（`isDream` 时 early return）。

### 4.3 其它阶段 objective 示例

| 阶段 | text | progress | target |
| --- | --- | --- | --- |
| R0 | 靠近发光的手机，按 E | 无 | 手机坐标 |
| R1 | 拾取：言、语 | 言/语字格 | 未获得碎片 |
| R2 | 靠近绿色梗鬼 | 无 | 梗鬼 |
| R3 | 靠近噪声终端 | 无 | 黑客怪 |
| 出口 | 走向金色裂隙醒来 | 无 | 裂隙 |

## 五、线性状态机（替换现有松散 notify）

```
step: intro → phone → collect → battle_menu → battle_hack → wake_gate → done
```

| step | 允许的操作 | 完成条件 | 下一 step |
| --- | --- | --- | --- |
| intro | 仅对话 | 对话结束且未 skip | phone |
| phone | 仅 R0 手机 E | phone 对话完 | collect（开门A，刷两字） |
| collect | 仅捡字 + 门B | 言且语 | battle_menu（开门B，刷常规怪） |
| battle_menu | 接触开战 | 战斗结束任意结果 | battle_hack（开门C，刷黑客） |
| battle_hack | 接触开战 | 战斗结束任意结果 | wake_gate（刷裂隙） |
| wake_gate | E 裂隙 | completeDreamOnboarding | done |

**硬约束：**

- 非当前 step 的 interactable / enemy **不生成或 `_locked: true` 交互提示「还不是时候」**。  
- 门用墙块动态增删，或 `scene_change` gate 绑定 step flag。  
- 跳过教学：任意时刻选项 / 设置 → `skipDreamOnboarding`（保留）。

## 六、战斗教学（仍线性）

### 6.1 常规战（R2）

- 开战前对话：顾言害怕 + 系统 1 句键位。  
- 敌人：弱血、低伤（已有 dream 数值可再降）。  
- 战后：不论胜负都推进（传说之下早期死了会重来——可选「败则 SAN 回满原地再战」或「败也开门但 toast 说明」）。  
- **推荐：** 败 → 弱化再战一次；第二次必放行，避免卡死。

### 6.2 黑客战（R3）

- 开战前对话：对比「菜单 vs 实时」。  
- `hackOpts` 保持短路由、低伤。  
- 战后才生成裂隙（防止跳过）。

### 6.3 不与主线抢 flag

继续用 `onboarding_battle_*` / `onboarding_hack_*`，**不要**写 `first_geng_intro_done` / `hack_core_intro_done`。

## 七、衔接剧情（序幕 → 梦 → 冷冻仓）

```
intro_3d 结束
  → 黑场 0.3s（可选）
  → dream R0 + 顾言「这是哪儿」
  → ……线性教学……
  → 裂隙
  → onboarding_wake（困惑落地）
  → freeze_center，hasClothes=false，objective=换衣服出门
```

**与旧 wake 的关系：**

- 走序幕+梦境：`wake_done=true`，**不再播**长 `DIALOGS.wake`。  
- 无序幕直接进（开发/读档）：不进梦；冷冻仓仍可走原 wake。  
- 醒来对白要补一句：**「刚才那些……是梦吗？」**，避免玩家以为还在教程世界。

## 八、UI / 手感细则

| 项 | 规格 |
| --- | --- |
| 同时高亮 | 仅 1 个交互物脉冲（金圈） |
| 对话框 | 全屏底栏，与正式一致；选项最多 2 个 |

| 底部 | 可保留世界 footer，但教学中可改成当前一步键位一句 |

## 十、验收标准

1. 玩家**无法**在未捡字时进入战斗房。  
2. 捡字时左上角**始终**显示「言」「语」两格及勾选状态。  
3. 开场 10 秒内玩家能感到「顾言不知道这是哪」，而不是被系统上课。  
4. 两种战斗**固定顺序**各经历一次后，才出现醒来裂隙。  
5. 醒来后无教学字、无教学击杀、目标为换衣服出门。  
6. 跳过教学仍能正常进冷冻中心。
