# 墓之语

> **你记得的词，是这个世界最后的声音。**

墓之语（muzhiyu）是一款基于 **Node.js + Vite + 原生 Canvas 2D** 构建的开放世界废墟探索 RPG。玩家扮演从冷冻仓醒来的**顾言**，在 2147 年的上海废墟中收集汉字、用诗词对抗「梗」污染、唤醒失语者，追寻良心 AI **Sydney（听雨）**与造物者**方知远**留下的记忆碎片，在慈悲与残忍之间走向六种结局。

> 项目使用 npm 管理依赖、Vite 作开发服务器与打包工具、Node.js 提供静态与 AI 代理服务。若计划接入 Vue 等前端框架，可在 `src/ui/` 的叠层 UI 层之上渐进式引入组件化方案（详见文末「技术栈」）。

## 故事

公元 2147 年，一场「大失语」摧毁了文明。高刺激短句与烂梗取代了完整的句子，语言崩塌后世界也跟着「塌平」——变成噪声与二维的废墟。

顾言是最后一批记得完整句子的冬眠者。苏醒时已物是人非。他捡起地上发光的汉字碎片，用诗词补全被梗吞噬的招牌与失语者的记忆，在与**梗鬼**的弹幕对决中做出选择：净化还是消灭，宽恕还是残忍。

江堤老人**守砚**赠予刻刀与诗的指引；数据中心尽头的蓝色光影**Sydney**在等待一个回答；散落在废图书馆、网络中枢与记忆深渊的三片记忆碎片，藏着造物者方知远最后的秘密。

## 核心玩法

### 三柱设计：诗 · 剑 · 衡

| 柱 | 含义 | 在游戏里你做什么 |
|----|------|------------------|
| **诗** | 收集与复原 | 捡汉字碎片 → 补全诗词 → 释放诗词大招 |
| **剑** | 以文化对抗暴力 | 探索劝退弱梗鬼 / 三种随机战斗 / Boss 弹幕 |
| **衡** | 道德与结局 | 净化 vs 消灭、宽恕 vs 攻击 → karma 驱动六结局 |

### 主循环

```text
看见梗化（招牌 / 失语者 / 梗鬼）
  → 捡汉字碎片
  → 靠近目标按 F 补全诗句（净化 / 唤醒）
  → 接触梗鬼进入随机战斗（弹幕 / 斩击 / 骇入）
  → 场景与 karma 反馈 → 推门禁、见 NPC、走向多结局
```

### F 补诗净化（Utterance）

游戏最核心的机制。靠近被梗污染的招牌或失语者，按 **F** 打开补诗界面，用已收集的汉字填入空缺。成功后：

- **梗墙净化**：热梗招牌恢复为正常路名，场景从扁平噪声「立」回三维
- **唤醒失语者**：只会复读碎片的人重新说出一句完整有情绪的话

### 三种随机战斗

每次接触梗鬼，系统在三种战斗模式中随机选择（尽量不连续重复）：

| 模式 | 玩法 |
|------|------|
| **UT 弹幕菜单战** | 菜单选择攻击/调查/净化/宽恕，弹幕阶段移动红心躲避，**K** 释放诗词大招 |
| **斩击·言锋对决** | WASD 移动，空格发射汉字顶回烂梗弹幕 |
| **骇入·逻辑空间** | 逻辑空间射击/躲避，被「格式化者」击中会暂时失去诗词碎片 |

- 消灭 → **残忍 +1**；净化/宽恕/救助 → **慈悲 +1**（右上角常显）
- 弱梗鬼可在探索中直接用汉字弹药劝退（E/F），无需进入战斗
- `enemy.forceCombat` / `enemy.combat` 可强制指定模式

## 章节流程

```text
标题页 → 3D 序幕（intro_3d.html）
  │
  ├─ 梦境教学：移动、捡字、三战教学 → 醒来
  ├─ 第一章 · 苏醒：冷冻中心 → 废弃街道
  │     收集《关雎》字（洲/逑）→ F 净化招牌、唤醒失语者
  ├─ 第二章 · 守砚：江堤横版 → 居民区
  │     横版跳跃/挥刀见守砚 → 收集《滕王阁序》《正气歌》字
  ├─ 第三章 · 茧房：体育馆
  │     算法迷宫、诗屏解谜削弱 Boss、推荐之核
  ├─ 第四章 · 终局：数据中心
  │     面对 Sydney → karma 决定主结局
  │     【火种 / 沉默 / 燃尽】
  │
  └─ 第五章 · 余烬（通关后可选）
        废图书馆 → 网络中枢 → 记忆深渊
        补全《将进酒》《春望》《月夜忆舍弟》
        收集三片记忆碎片 → 方知远的真相
        【造物者的忏悔 / 永恒回响 / 文字花园（隐藏）】
```

### 支线

- **失语者村落**：5 位失语者各有不同的缺词与醒来后的话，全部唤醒后长出「文字树」
- **方知远的日记**：散落在三个余烬区域的 12 页日记，拼凑造物者的完整故事
- **要石刻字**：在要石/残碑上留下自定义文字，AI 可点评（无密钥则跳过）
- **言锋试炼**：标题页的无尽模式，挑战连续战斗

## 功能特性

| 特性 | 说明 |
|------|------|
| **F 补诗净化** | 核心机制，净化梗墙与唤醒失语者 |
| **三种随机战斗** | UT 弹幕 / 斩击言锋 / 骇入逻辑空间 |
| **江堤横版关卡** | 跳跃、挥刀、踩踏梗鬼 |
| **多结局系统** | karma + 探索深度 → 6 种结局 |
| **AI 叙事导演** | 关键 NPC 动态生成分支对白（无密钥自动降级为静态剧情） |
| **Sydney 自由对话** | 终局自由对话 + 快捷句，注入玩家历程，个性化 epilogue |
| **TTS 语音** | 角色配音（可降级为纯文字） |
| **要石存档** | 存档点 + 自动存档 + 手动槽位 + 快速存读 |
| **难度系统** | 简单/普通/困难，影响敌人生命、弹幕与 SAN |
| **小地图** | 战争迷雾式探索地图（Tab 切换） |
| **任务/背包/地图** | Q/I/M 面板 |
| **刻字系统** | 在要石上留下自定义文字 |
| **无尽模式** | 标题页「言锋试炼」 |
| **移动端触控** | 虚拟摇杆 + 触屏按钮 |
| **无障碍** | 高对比字幕、降低特效、音频倍速、对话速度 |

功能开关见 `src/config.js` 的 `FEATURES`（`level3d` 默认关闭，为调试遗留的 3D 维度裂隙关卡）。

## 运行方式

需要 Node.js 18+。

```bash
npm install
npm run dev          # Vite + AI/静态服务
npm start            # 仅 Node 服务 → http://localhost:8080
```

兼容入口：`node server.js`。改端口：`PORT=9000 npm start`。

```bash
npm run build
npm test
npm run lint
npm run format:check
```

## AI 接入（可选）

同源服务提供 LLM / TTS 代理；**无密钥可完整通关**，自动降级为纯文字与静态剧情。

- **叙事导演**：中段关键 NPC（守砚、失语者、茧房等）可生成分支，写入线索/态度/世界史（`StoryState`，可存档）
- **Sydney 终局**：自由对话 + 快捷句，注入玩家历程与线索，个性化 epilogue；TTS 配音保留
- 配置：`ai_keys.local.json`（模板 `ai_keys.example.json`）
- 或环境变量：`DEEPSEEK_API_KEY`、`MIMO_API_KEY`
- 接口：`GET /api/health`、`POST /api/llm`、`POST /api/tts`
- 预热 TTS：`node scripts/prewarm.mjs`
- 开关：`src/config.js` → `FEATURES.aiDirector`

## 技术栈

- **运行环境**：Node.js 18+（npm 管理依赖）
- **渲染**：HTML5 Canvas 2D（高 DPI 适配）+ WebGL（3D 调试关卡）
- **构建 / 开发服务器**：Vite（HMR、打包、API 代理）
- **服务**：Node.js 原生 HTTP（静态资源 + AI 代理，见 `server/`）
- **测试**：Vitest
- **代码规范**：ESLint + Prettier
- **AI**：DeepSeek / MiMo LLM + 浏览器 Web Speech TTS

> 关于「Vue / Node / npm」技术栈：本项目已完整覆盖 Node + npm + Vite 的现代前端工程化能力，核心玩法以原生 Canvas 渲染（无框架开销）为主。如需引入 Vue 组件化 UI，建议从 `src/ui/`（叠层与代币 UI）与标题页/结算页这类「非实时战斗」界面切入，避免框架重渲染干扰游戏主循环。
