# 墓之语

《墓之语》是一款 HTML5 Canvas 2D 废墟探索 RPG。玩家扮演从冷冻仓醒来的顾言，在 2147 年的上海废墟中收集诗词碎片、对抗烂梗污染、唤醒失语者，并追寻良心 AI「Sydney」与方知远留下的记忆。

当前版本包含主线章节、第五章「余烬」、江堤横版关卡、维度裂隙 3D 关卡、刻字、背包、任务列表、存档、难度、小地图、移动端触控按钮、诗词大招、AI 对话与 TTS 配音。

## 游戏内容

- **探索**：冷冻中心、废弃街道、旧地铁站、黄浦江江堤、居民区、体育馆、数据中心，以及第五章的废图书馆、网络中枢、记忆深渊、失语者聚居地。
- **战斗**：回合制弹幕战斗。玩家用红心躲避烂梗弹幕，也可以用刻刀攻击、调查、诗词碎片净化或宽恕敌人。
- **诗词谜题**：收集散落汉字，复原《关雎》《春晓》《将进酒》《春望》《月夜忆舍弟》等诗句，打开场景门禁或解锁记忆碎片。
- **第五章「余烬」**：找回 Sydney 的三片核心记忆，在巨大要石前做出终章选择，并在数据中心完成对应终局。
- **刻字与要石**：要石可作为复活点，也能留下自定义刻痕。AI 可在结局处汇总玩家刻下的文字。
- **支线**：唤醒失语者、收集旧书页和语言种子、阅读方知远日记。
- **移动端**：触屏设备会显示虚拟摇杆、交互按钮、冲刺/确认按钮和诗词大招按钮。

## 运行方式

需要 Node.js 18 或更高版本。

```bash
npm install
```

开发模式：

```bash
npm run dev
```

该命令会同时启动 Node AI/静态服务器和 Vite 开发服务器。通常打开 Vite 输出的本地地址即可。

试玩/服务端模式：

```bash
npm start
```

默认地址为：

```text
http://localhost:8080
```

也可以使用兼容入口：

```bash
node server.js
```

修改端口：

```bash
PORT=9000 npm start
```

## AI 接入

完整体验需要同源 Node 服务提供 AI 代理：

- **LLM 导演**：关键剧情分支和 Sydney 自由对话使用 DeepSeek 接口。
- **TTS 配音**：对白可通过小米 MiMo TTS 生成语音。
- **语音缓存**：生成后的音频会缓存到 `cache/tts/`，避免重复请求。

密钥配置文件：

```text
ai_keys.local.json
```

该文件已被 `.gitignore` 忽略。模板见：

```text
ai_keys.example.json
```

也可以使用环境变量：

```bash
DEEPSEEK_API_KEY=xxx
MIMO_API_KEY=xxx
```

如果没有配置 AI，前端会自动降级为纯文字、静态剧情和静态结局，不会卡死。

### 服务接口

- `GET /api/health`：返回 AI 可用状态。
- `POST /api/llm`：代理 LLM 对话请求。
- `POST /api/tts`：生成或读取缓存语音，返回 PCM16 音频流。

### 预热语音缓存

启动服务器后可预生成静态对白语音：

```bash
node scripts/prewarm.mjs
```

指定端口：

```bash
PORT=9000 node scripts/prewarm.mjs
```

## 系统说明

### 存档

- 自动存档：场景切换、要石激活、战斗胜利或宽恕、通关等节点。
- 手动存档：`F6` 打开存档菜单，支持 3 个手动槽位。
- 快速存读：`F5` 快速存档，`F9` 快速读档。
- 存档包含场景、坐标、flags、karma、收集物、敌人击败记录、要石、刻字、难度和小地图探索记录。

### 战斗

- 战斗菜单包含战斗、调查、诗词、宽恕。
- 攻击瞄准使用 `E` 或 `Space` 停下光标。
- 诗词大招需要先收集完整诗句，每场战斗限用一次。
- 部分敌人具有特殊弹幕，例如格式化者会临时删除已收集的诗词碎片。

### 难度

三档难度会影响敌人生命、弹幕密度、弹幕速度、SAN 伤害、SAN 上限和宽恕所需调查次数。

### 小地图

右上角小地图支持战争迷雾，显示已探索区域、玩家、目标、道具、要石、NPC 和敌人。

## 项目结构

```text
.
├─ index.html              # 主游戏入口
├─ intro_3d.html           # 序幕 3D 页面
├─ server.js               # 兼容入口
├─ server/                 # 静态服务与 AI 代理
├─ scripts/                # 工具脚本，如 TTS 预热
├─ src/
│  ├─ game.js              # 游戏主状态机
│  ├─ scenes.js            # 场景数据、敌人和交互点
│  ├─ render.js            # 总渲染入口
│  ├─ render/              # HUD、场景、对话、战斗、面板渲染
│  ├─ systems/             # 交互、对话、刻字等系统
│  ├─ data/                # 对话、谜题、场景切换文案
│  ├─ ai/                  # LLM/TTS 前端接入
│  ├─ battle.js            # 战斗系统
│  ├─ sidescroll.js        # 江堤横版关卡
│  └─ level3d.js           # 维度裂隙 3D 关卡
├─ tests/                  # Vitest 测试
├─ assets/                 # 音频等资源
├─ docs/                   # 设计与剧情文档
└─ cache/tts/              # TTS 缓存目录
```
